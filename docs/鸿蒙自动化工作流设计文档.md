**表 1：触发条件清单 (Triggers) —— “当发生什么时...”**

设计重点：强化\*\*“鸿蒙分布式”\*\*感知，不仅是手机本身，而是整个鸿蒙生态设备。

| 一级分类 | 二级分类/触发器 | 参数/配置项 | 典型用户场景 (User Story) | 鸿蒙特性 |
| :---- | :---- | :---- | :---- | :---- |
| **基础环境** | **时间/日期** | 具体时间、日出/日落、工作日/节假日 | 每天早上7点自动播报天气。 |  |
|  | **地理位置** | 到达/离开某地、半径范围 | **离开公司**半径500米时，自动发消息给家人。 |  |
|  | **Wi-Fi 状态** | 连接/断开指定SSID | 当连接到**家中Wi-Fi**时，手机自动静音。 |  |
|  | **蓝牙连接** | 连接/断开指定设备 | 当连接**车载蓝牙**时，自动开始播放音乐。 |  |
| **设备状态** | **电源/电量** | 充电中、断电、电量 \< X% | 电量低于20%时，自动开启省电模式并降低亮度。 |  |
|  | **应用状态** | 打开/关闭指定App | 打开\*\*“抖音”\*\*时，自动将媒体音量调至30%（防吓到人）。 |  |
|  | **NFC 标签** | 扫描特定 Tag ID | 手机**碰一碰**床头贴纸，执行“助眠模式”。 | 🌟 一碰传心智 |
|  | **剪贴板** | 包含特定关键词/正则匹配 | 复制包含“快递单号”文本时，弹窗提示查询。 |  |
| **鸿蒙生态** | **超级终端** | 设备接入/断开 | 当**MatePad**接入超级终端时，自动流转当前视频。 | 🌟 分布式软总线 |
|  | **穿戴设备** | 睡眠状态、心率阈值、锻炼状态 | 手表监测到**用户睡醒**，自动打开窗帘（IoT联动）。 | 🌟 运动健康 |
|  | **Hicar (车机)** | 车辆启动、熄火、连接Hicar | **车辆熄火**断开Hicar时，自动记录停车位置。 | 🌟 智慧出行 |
|  | **智能家居** | 设备状态变化（开/关/数值） | 当**智能门锁**被指纹解锁时，客厅灯光亮起。 | 🌟 Harmony Connect |
| **用户意图** | **桌面卡片** | 点击卡片特定按钮 | 点击桌面的“一键打车”**元服务卡片**。 | 🌟 原子化服务 |
|  | **语音唤醒** | 特定指令词 | 对小艺说“**我要开会了**”，执行勿扰+录音流程。 | 🌟 小艺融合 |
|  | **手势/物理** | 摇一摇、背部轻敲 | **剧烈摇动**手机，快速弹出付款码。 |  |

**表 2：动作节点清单 (Actions) —— “...就执行什么”**

设计重点：覆盖**系统底层能力**与**中国互联网特色服务**（微信/支付宝/高德）。

| 一级分类 | 动作名称 | 动作参数/细节 | 典型用途 |
| :---- | :---- | :---- | :---- |
| **系统控制** | **无线管控** | 开关 Wi-Fi / 蓝牙 / 5G / 飞行模式 / 热点 | 离家自动关Wi-Fi省电。 |
|  | **显示/外观** | 亮度调节、深色模式、息屏显示、锁定屏幕 | 晚上看书自动调暖色温。 |
|  | **音量/通知** | 铃声/媒体音量、震动模式、朗读文本 (TTS) | 结合TTS播报早安新闻。 |
|  | **实用工具** | 截图、录屏、开启手电筒、倒计时 | 快速记录屏幕操作。 |
| **应用服务** | **微信 (WeChat)** | 扫一扫、付款码、发送预设文本、打开朋友圈 | **一键直达付款码**，省去查找步骤。 |
| (本地化) | **支付宝 (Alipay)** | 乘车码、健康码、收能量、转账 | 早上通勤一键开乘车码。 |
|  | **导航 (Map)** | 发起导航(回家/公司)、查询路况、周边搜索 | 上车自动导航回家。 |
|  | **媒体 (Media)** | 播放/暂停、下一首、打开特定歌单 | 连接耳机自动放歌。 |
| **跨端流转** | **任务流转** | 发送当前URL/文件到指定设备(TV/Pad/PC) | 手机没看完的文章**一键甩到平板**上接着看。 |
| (鸿蒙核心) | **远程调用** | 调用其他设备摄像头/麦克风 | 用手机查看家里智慧屏摄像头的画面。 |
|  | **跨端剪贴板** | 获取/设置云剪贴板内容 | 手机复制，电脑直接粘贴。 |
|  | **Hicar 控制** | 开空调、开关窗、寻车、解锁 | 下楼前提前热车/开空调。 |
| **文件处理** | **文件操作** | 保存文件、压缩/解压、转换格式(如PDF转图) | 处理工作文档。 |
|  | **相册操作** | 保存图片到相册、删除最近照片、获取最新照片 | 自动化整理截图。 |

**表 3：逻辑与数据处理 (Logic & Core) —— “流程的大脑”**

这部分决定了你的App是“玩具”还是“生产力工具”。

| 节点类型 | 节点名称 | 功能描述 | 开发者备注 (Dev Note) |
| :---- | :---- | :---- | :---- |
| **流程控制** | **如果 (If / Else)** | 判断条件是否满足（如：如果现在是晚上） | 支持嵌套 |
|  | **重复 (Repeat)** | 循环执行某段动作 X 次 |  |
|  | **等待 (Wait)** | 暂停流程 X 秒/分钟 | 这里的保活机制是技术难点 |
|  | **退出 (Exit)** | 强制结束当前工作流 |  |
|  | **菜单 (Menu)** | 弹出一个选项菜单供用户选择 | 用户交互的关键节点 |
| **变量/数据** | **设置变量** | 存储文本、数字或文件路径 |  |
|  | **获取输入** | 弹窗请求用户输入文本/数字 |  |
|  | **计算器** | 进行加减乘除运算 |  |
| **高级能力** | **网络请求 (HTTP)** | GET / POST / PUT 请求，支持自定义Header | **Pro版功能**，给极客用户连接API用 |
|  | **正则表达式** | 文本提取、替换 | 用于处理复杂的网页或文本数据 |
|  | **JSON 解析** | 解析网络请求返回的数据字典 | 配合HTTP请求使用 |
|  | **运行脚本** | 支持简单的 JavaScript 或 Python (可选) | 极客的终极需求 (如类似于JSBox) |
